# 26QGSummerCamp

## 仓库说明

本仓库用于提交和管理夏令营期间的个人笔记。
每位成员只维护自己的目录，并将每周周记上传到对应的周次目录下。

## 目录规范

1. 每位成员在仓库根目录下创建一个属于自己的一级目录。
2. 一级目录名称统一使用本人中文姓名。
3. 个人目录下按周次创建子目录，例如 `第一周`、`第二周`。
4. 每周内容使用 `.md` 文件记录，并放在对应周目录下。

## 周记内容要求

每周的周记内容统一分为以下四个部分：

1. `生活随记`
   记录本周生活中的见闻、感受、碎片化思考或值得一提的小事。
2. `学习开发情况`
   记录本周学习内容、项目推进、代码实践、问题排查和开发收获。
3. `一周总结`
   总结本周完成了什么、有哪些进步、哪些目标达成或未达成。
4. `存在问题未来规划`
   说明当前存在的问题、不足和下周的改进方向、学习计划或开发安排。

建议每周周记都按照以上四个标题展开，便于统一查看和后续整理。

## 目录示例

```text
26QGSummerCamp/
├── 张三/
│   ├── 第一周/
│   │   └── 周记.md
│   └── 第二周/
│       └── 周记.md
└── 李四/
    ├── 第一周/
    │   └── 周记.md
    └── 第二周/
        └── 周记.md
```

## 周记模板示例

```md
# 第一周周记

## 生活随记

## 学习开发情况

## 一周总结

## 存在问题未来规划
```

## 注意事项

1. 只在自己的目录下新增或修改内容，不要改动他人的目录和笔记。
2. 新成员请先按自己的中文姓名创建一级目录，再按周次上传周记。
3. 上传的笔记文件统一使用 `.md` 格式。
